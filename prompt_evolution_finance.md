# 💰 財經模式 (Finance Mode) 提示詞演進史 (v0.0.1 -> v0.0.15)

本文件旨在以「檔案比較」的形式，精確追蹤 `Daily_Cyber_Fin` 專案中，針對三種財經核心 AI 輸出：**財經早報**、**財經 TG 教學**、**財經 MD 日報** 的提示詞 (Prompt) 的完整演變過程。

---

## 1. 📈 財經早報 (Morning Forecast)

此提示詞的目標是生成每日台股開盤前的「美股收盤與台股開盤預測」，追求**快速**、**重點導向**。

### v0.0.2 - v0.0.15
> **[結構穩定]** 此提示詞從 v0.0.2 版加入後，其結構一直保持極度穩定，是專案中最長壽的提示詞之一。它簡單、有效，完美專注於「盤前速報」這一個核心任務，因此無須任何修改。
```
華爾街操盤手簡報。新聞:{raw_us}\n關注:{targets}\n任務:美股收盤簡報+台股開盤預測。\n要求: 繁體中文, 純文字, Emoji 排版。
```

## 2. 📢 財經 TG 教學 (Finance Telegram Class)

此提示詞的目標是生成每日盤中或盤後的「市場分析與 K 線教學」，追求**實用性**與**教學價值**。

### v0.0.1
> 最初版本，指令較為模糊，將市場分析與教學混在一個任務中。
```
你是 CFO 與技術導師。
【持股】{tech_str}
【新聞】{raw_news}
【任務】
1. **使用繁體中文**。
2. 對持股進行利多/利空分析。
3. K 線教學：解釋一個重要的 K 線型態 (例如：黃金交叉, W 底, 頭肩頂)。
4. **嚴禁使用 Markdown 粗體**。
```

### v0.0.4
> **[關鍵演進]** 首次將任務明確拆分為「市場脈動」與「K 線教學」兩部分，並為教學部分設計了固定的結構，大幅提升了輸出的穩定性與品質。

## 3. 📄 財經 MD 日報 (Finance Markdown Log)

此提示詞的目標是生成一份**極度詳盡、數據驅動**的 Markdown 報告，專門用於訓練 AI 知識庫 (如 Google NotebookLM)。

### v0.0.1
> 最初版本，僅有一個模糊的指令，未定義任何結構。
```
完整的投資日報 (繁中)
{tech_str}
{raw_news}
```

### v0.0.2
> **[關鍵演進]** 第一次為 MD 日報設計了詳細結構，包含持股分析、新聞解讀與技術教學，是從 0 到 1 的飛躍。
```
請撰寫一份**完整的投資策略日報 (Markdown 格式)**。
【日期】{today_str}
【持股數據】
{tech_str}
【市場新聞】
{raw_news}

【要求】
1. 使用豐富的 Markdown 排版 (標題、列表、粗體)。
2. **持股分析**：請詳細列出每一檔股票的利多、利空因素。
3. **新聞解讀**：請引用新聞並分析對台股的影響。
4. **技術教學**：請詳細解說今日的 K 線型態教學。
```

### v0.0.4
> **[關鍵演進]** 引入「ASCII Art 圖解」和「交易員心得筆記」，讓報告更具視覺化與個人洞見。
```diff
  ## 1. 📈 持股深度診斷
  - 請逐一分析持股，並給出具體操作建議 (Buy/Sell/Hold)。
  - 標示每則新聞對特定持股是「🔴 利空」還是「🟢 利多」。
  
  ## 2. 🕯️ K 線技術教學 (含圖解)
- - 請嘗試用 ASCII Art 畫出今日教學的 K 線型態 (例如紅K、十字線)。
+ - 請嘗試用 ASCII Art 畫出今日教學的 K 線型態 (例如紅K、十字線)。
  - 詳細解釋技術指標 (MA, RSI, MACD) 的運用。
  
- ## 3. 📝 交易員心得筆記
- - 總結今日盤勢與明日觀察重點。
+ ## 3. 📝 交易員心得筆記
+ - 總結今日盤勢與明日觀察重點。
```

### v0.0.7
> **[風格擺盪]** 追求專業化，引入「華爾街分析師」角色，要求使用 Markdown 表格，並禁止使用 Emoji，使風格轉向嚴肅、數據導向。

### v0.0.9 - v0.0.15
> **[結構穩定]** 確立了最終方向：明確告知 AI 「此文件將用於知識庫訓練」，並融合了先前版本的優點：使用 Markdown 表格進行深度診斷、用 ASCII Art 進行圖解教學、以及交易員的心得筆記。此結構在 v0.0.9 後便穩定下來，後續版本僅針對程式穩定性進行優化，提示詞本身未再變動。
